# 登录注册接口文档

本文档基于当前仓库代码整理，面向前端联调。

## 1. 说明

- 服务默认启动端口：`8080`
- 推荐前端优先调用公开接口
- 当前不存在 `/luoluo/api/auth/register` 这类路径
- 当前真实接口路径以 `/luoluo/system/public/member/auth/...` 和 `/luoluo/member/auth/...` 为准

## 2. 推荐使用的接口

未登录场景，推荐前端统一调用这一组：

- 发送验证码：`POST /luoluo/system/public/member/auth/code/send`
- 登录：`POST /luoluo/system/public/member/auth/login`
- 注册：`POST /luoluo/system/public/member/auth/register`
- 图形验证码：`GET /luoluo/system/public/captcha/image`

会员模块内也提供一组接口：

- 发送验证码：`POST /luoluo/member/auth/send-code`
- 登录：`POST /luoluo/member/auth/login`
- 注册：`POST /luoluo/member/auth/register`
- 登出：`POST /luoluo/member/auth/logout`

## 3. grantType 说明

`grantType` 表示登录/注册方式，不是用户身份。

可选值：

- `password`：账号密码方式
- `sms`：手机验证码方式
- `email`：邮箱验证码方式

注意：

- 登录时必须传 `grantType`
- 注册时必须传 `grantType`
- 前端不用传普通用户类型
- 后端注册成功后默认写入 `userType = USER`

## 4. 统一响应格式

大多数接口返回结构如下：

```json
{
  "code": 0,
  "message": "success message",
  "data": {}
}
```

失败时通常返回：

```json
{
  "timestamp": "2026-04-15T20:00:00.000+08:00",
  "status": 400,
  "error": "Bad Request",
  "message": "grantType must not be blank",
  "path": "/luoluo/system/public/member/auth/register"
}
```

## 5. 图形验证码接口

### 5.1 获取图形验证码

`GET /luoluo/system/public/captcha/image`

用途：

- 获取图形验证码图片
- 获取 `captchaKey`

响应示例：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "captchaKey": "8df3f4c1f2b54f0f9f2f9c5f0d8a1234",
    "imageBase64": "PHN2ZyB4bWxu...",
    "ttlSeconds": 120
  }
}
```

前端图片可直接渲染为：

```text
data:image/svg+xml;base64,${imageBase64}
```

## 6. 发送验证码接口

### 6.1 公开发送验证码

`POST /luoluo/system/public/member/auth/code/send`

请求字段：

- `grantType`：必填，`sms` 或 `email`
- `bizType`：可选，建议注册传 `REGISTER`，登录传 `LOGIN`
- `captchaKey`：当启用图形验证码时必填
- `captchaCode`：当启用图形验证码时必填
- `phone`：`grantType=sms` 时必填
- `email`：`grantType=email` 时必填

邮箱注册发码示例：

```json
{
  "grantType": "email",
  "bizType": "REGISTER",
  "captchaKey": "8df3f4c1f2b54f0f9f2f9c5f0d8a1234",
  "captchaCode": "2DM4",
  "email": "2130265486@qq.com"
}
```

手机注册发码示例：

```json
{
  "grantType": "sms",
  "bizType": "REGISTER",
  "captchaKey": "8df3f4c1f2b54f0f9f2f9c5f0d8a1234",
  "captchaCode": "2DM4",
  "phone": "17328166785"
}
```

成功响应示例：

```json
{
  "code": 0,
  "message": "send code success",
  "data": {
    "requestId": "req-123456",
    "grantType": "email",
    "maskedTarget": "2***@qq.com",
    "ttlSeconds": 120,
    "debugVerifyCode": null
  }
}
```

说明：

- 同一目标默认 `60` 秒内不能重复发送
- 当前配置里启用了图形验证码校验
- `bizType` 不传时后端默认按 `LOGIN` 处理

## 7. 登录接口

### 7.1 公开登录

`POST /luoluo/system/public/member/auth/login`

### 7.2 会员登录

`POST /luoluo/member/auth/login`

二者底层逻辑一致。

请求公共字段：

- `grantType`：必填
- `deviceType`：可选，例如 `web`、`h5`、`app`

### 7.3 密码登录

请求示例：

```json
{
  "grantType": "password",
  "deviceType": "web",
  "username": "demo_user",
  "password": "123456"
}
```

说明：

- `username`、`email`、`phone` 三者实际可任选一个作为密码登录账号
- 只要后端能匹配到账户即可

### 7.4 手机验证码登录

请求示例：

```json
{
  "grantType": "sms",
  "deviceType": "web",
  "phone": "17328166785",
  "smsCode": "123456"
}
```

### 7.5 邮箱验证码登录

请求示例：

```json
{
  "grantType": "email",
  "deviceType": "web",
  "email": "2130265486@qq.com",
  "emailCode": "123456"
}
```

### 7.6 登录成功响应

```json
{
  "code": 0,
  "message": "login success",
  "data": {
    "token": "xxxxxxxxxxxxxxxx",
    "tokenType": "Bearer",
    "expiresIn": 86400,
    "grantType": "email",
    "user": {
      "userId": 1,
      "username": "demo_user",
      "displayName": "落落",
      "phone": "17328166785",
      "email": "2130265486@qq.com",
      "userType": "USER"
    }
  }
}
```

前端后续请求头示例：

```text
Authorization: Bearer xxxxxxxxxxxxxxxx
```

## 8. 注册接口

### 8.1 公开注册

`POST /luoluo/system/public/member/auth/register`

### 8.2 会员注册

`POST /luoluo/member/auth/register`

二者底层逻辑一致。

请求公共字段：

- `grantType`：必填
- `deviceType`：可选
- `password`：必填
- `confirmPassword`：可选，传了就必须和 `password` 一致
- `displayName`：可选

### 8.3 密码注册

适用场景：账号密码直接注册。

请求示例：

```json
{
  "grantType": "password",
  "deviceType": "web",
  "username": "luoluo",
  "password": "123456",
  "confirmPassword": "123456",
  "displayName": "落落"
}
```

字段要求：

- `username` 必填
- 不需要验证码

### 8.4 手机验证码注册

适用场景：手机号注册。

请求示例：

```json
{
  "grantType": "sms",
  "deviceType": "web",
  "phone": "17328166785",
  "smsCode": "181019",
  "password": "123456",
  "confirmPassword": "123456",
  "displayName": "落落"
}
```

字段要求：

- `phone` 必填
- `smsCode` 必填
- 如果不传 `username`，后端默认把手机号作为用户名

### 8.5 邮箱验证码注册

适用场景：邮箱注册。

请求示例：

```json
{
  "grantType": "email",
  "deviceType": "web",
  "email": "2130265486@qq.com",
  "emailCode": "181019",
  "phone": "17328166785",
  "password": "123456",
  "confirmPassword": "123456",
  "displayName": "落落"
}
```

字段要求：

- `email` 必填
- `emailCode` 必填
- `phone` 非必填
- 如果不传 `username`，后端默认把邮箱作为用户名

### 8.6 注册成功响应

注册成功后会直接返回登录态，不需要再单独调一次登录接口。

```json
{
  "code": 0,
  "message": "register success",
  "data": {
    "token": "xxxxxxxxxxxxxxxx",
    "tokenType": "Bearer",
    "expiresIn": 86400,
    "grantType": "email",
    "user": {
      "userId": 12,
      "username": "2130265486@qq.com",
      "displayName": "落落",
      "phone": "17328166785",
      "email": "2130265486@qq.com",
      "userType": "USER"
    }
  }
}
```

## 9. 登出接口

`POST /luoluo/member/auth/logout`

请求头：

```text
Authorization: Bearer xxxxxxxxxxxxxxxx
```

成功响应：

```json
{
  "code": 0,
  "message": "logout success",
  "data": null
}
```

## 10. 常见错误

### 10.1 `grantType must not be blank`

原因：

- 前端没有传 `grantType`

解决：

- 邮箱注册传 `grantType: "email"`
- 手机注册传 `grantType: "sms"`
- 账号密码注册传 `grantType: "password"`

### 10.2 `No static resource luoluo/api/auth/register`

原因：

- 调用了不存在的路径 `/luoluo/api/auth/register`

解决：

- 改为调用真实接口 `/luoluo/system/public/member/auth/register`

### 10.3 `username already exists`

原因：

- 用户名已存在

### 10.4 `phone already exists`

原因：

- 手机号已被注册

### 10.5 `email already exists`

原因：

- 邮箱已被注册

### 10.6 `smsCode is invalid or expired`

原因：

- 手机验证码错误或已过期

### 10.7 `emailCode is invalid or expired`

原因：

- 邮箱验证码错误或已过期

## 11. 前端接入建议

- 邮箱注册页默认提交 `grantType: "email"`
- 手机注册页默认提交 `grantType: "sms"`
- 纯账号密码注册页默认提交 `grantType: "password"`
- 发送注册验证码时建议传 `bizType: "REGISTER"`
- 登录页和注册页不要共用错误的旧路径 `/luoluo/api/auth/...`
